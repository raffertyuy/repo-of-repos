name: analyze-repositories
description: Scan all repos in repos/ and update repos/repos.md with a description of each
user-invocable: true
---

Analyze every directory in `repos/` and update `repos/repos.md` with a summary of each repository.

For each directory in `repos/`:

1. Read its README.md (or README) if one exists
2. Check its `package.json`, `Cargo.toml`, `go.mod`, `pyproject.toml`, or equivalent to understand the tech stack
3. Read its git remote URL (`git -C repos/<name> remote get-url origin`)
4. Briefly look at the top-level directory structure

Then write `repos/repos.md` with this format:

```markdown
# Repositories

This file describes each repository cloned into `repos/`. It is auto-generated by the `/analyze-repositories` skill. Do not edit manually.

## <repo-name>

- **Remote**: <git remote URL>
- **Tech stack**: <languages, frameworks, key dependencies>
- **Description**: <1-2 sentence summary from README or inferred from code>
- **Key paths**: <notable top-level directories and what they contain>
- **Agent instructions**: <list which of CLAUDE.md, AGENTS.md, .github/copilot-instructions.md exist, or "None">

(repeat for each repo)
```

If `repos/` contains no directories, write the file with a placeholder comment indicating no repos were found.
