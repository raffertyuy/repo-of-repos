# Update All Repos

Fetch and pull the latest changes for every sub-repo in `repos/`, then re-analyze and update `repos/repos.md`.

## Order of Operations

1. **Pull all sub-repos** — iterate through each directory in `repos/*/` that is a git repository
2. **Re-analyze** — update `repos/repos.md` following the `/analyze-repositories` skill format

## For Each Sub-Repo

For each repo under `repos/<name>/`:

1. Verify it is a git repository (has a `.git` directory)
2. Run `git -C repos/<name> fetch --all --prune` to fetch all remotes and clean up stale tracking branches
3. Check the current branch: `git -C repos/<name> branch --show-current`
4. If the branch has an upstream, run `git -C repos/<name> pull --ff-only`
   - If fast-forward fails (diverged history), **do not force-pull** — report it and move on
   - If there are uncommitted local changes that block the pull, report it and move on
5. Report the result: updated (with new commit count), already up to date, or error

## Re-Analyze repos.md

After all sub-repos are updated, regenerate `repos/repos.md` by following the `/analyze-repositories` skill instructions:

For each directory in `repos/`:

1. Read its README.md (or README) if one exists
2. Check its `package.json`, `Cargo.toml`, `go.mod`, `pyproject.toml`, or equivalent to understand the tech stack
3. Read its git remote URL (`git -C repos/<name> remote get-url origin`)
4. Briefly look at the top-level directory structure
5. Check which agent instruction files exist: `CLAUDE.md`, `AGENTS.md`, `.github/copilot-instructions.md`

Then write `repos/repos.md` with this format:

```markdown
# Repositories

This file describes each repository cloned into `repos/`. It is auto-generated by the `/analyze-repositories` skill. Do not edit manually.

## <repo-name>

- **Remote**: <git remote URL>
- **Tech stack**: <languages, frameworks, key dependencies>
- **Description**: <1-2 sentence summary from README or inferred from code>
- **Key paths**: <notable top-level directories and what they contain>
- **Agent instructions**: <list which of CLAUDE.md, AGENTS.md, .github/copilot-instructions.md exist, or "None">

(repeat for each repo)
```

If `repos/` contains no directories, write the file with a placeholder comment indicating no repos were found.

## Summary

Report a summary at the end:

| Repo | Status |
|------|--------|
| `<name>` | Updated (N new commits) / Already up to date / Error: <reason> |

And note whether `repos/repos.md` was changed.

## Notes

- Never force-pull or reset sub-repos — if a pull fails, report and skip
- Never discard uncommitted local changes — report them as a blocker
- If a sub-repo has no remote configured, skip it with a warning
- Run fetches and pulls sequentially to avoid credential prompt conflicts
